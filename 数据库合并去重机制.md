📝 数据库合并去重机制
目标： 在不改变 Excel 原始数据的前提下，优化导出至计算器数据库（JSON）的逻辑，解决“参数相同但型号细微差别造成的干扰项”问题。

逻辑规则：

分组键 (Match Key)： 检查每行数据，凡是满足以下四个参数 完全一致 的型号，视为“同一计算分类”：

Standby Current (mA) (静态电流)
Alarm Current (mA) (报警电流)
ledCost (LED耗电标志)
type (设备大类，如 Detector, IO module, Sounder)
合并策略 (Merging Strategy)：

型号合并 (Name)： 把同组型号用 / 或 ( ) 拼接。
例如：600-001/003/005 或 600系列 (001, 003, 005)

描述合并 (Description)： 提取共同核心词，去除冗余后缀（如 Weatherproof, Red, White, Isolator 等），仅保留核心功能描述。
例如：将“单烟探测器”和“单烟带隔离探测器”合并为“单烟探测器 (包含隔离版)”

典型合并参考案例：

Detector: 合并 600-001, 003, 005 为 600系列探测器 (001/003/005)；合并 SNA/HNA 360 系列的同类参数。
IO Module: 忽略 Mains, Isolator 等后缀，按“输入输出路数”合并。如：626系列双路模块 (004/008/011/012)。
Sounder: 忽略 Weatherproof, Red, White 等，按电流值合并型号。
UI 展示优化：

如果同组包含超过 3 个型号，建议缩写为：前缀-起始号...结束号。
确保合并后的数据条目在 UI 下拉框中是唯一的，点击一次即可代表该类所有硬件。
